*2025년 3월 기준으로 공개된 정보를 기반으로 작성했습니다.*

1편에서는 현대 LLM의 기반이 되는 아키텍처 혁신들을 다뤘다. Flash Attention, RoPE, MoE — 모델을 더 크고 효율적으로 만드는 기술들.

그런데 더 크고 효율적인 모델이 더 좋은 모델은 아니다.

GPT-3는 강력했지만, 원하는 대로 동작하지 않았다. 사실을 유창하게 꾸며냈고, 때로는 위험한 내용을 자연스럽게 출력했다. 크기가 클수록 더 설득력 있게 틀렸다.

이 문제를 <strong>정렬(Alignment)</strong>이라 부른다. 모델이 우리가 원하는 방향으로 행동하게 만드는 것. 이번 편은 그 기술의 진화를 다룬다.

---

## 왜 정렬이 필요한가 — 목표 불일치 문제

언어 모델의 사전학습 목표는 단순하다. 다음 토큰을 예측하라. 이 목표로 학습하면 인터넷에 있는 글의 분포를 잘 따르는 모델이 된다.

문제는, 인터넷에는 거짓말도 있고, 혐오 발언도 있고, 위험한 정보도 있다는 것이다. "다음 토큰을 잘 예측하는 모델"이 곧 "유용하고 안전한 응답을 생성하는 모델"은 아니다.

이것이 **목표 불일치(goal misspecification)** 문제다. 우리가 실제로 원하는 것(helpful, harmless, honest)과 모델이 최적화하는 것(next-token prediction loss 최소화) 사이의 간극.

정렬 연구는 이 간극을 메우는 방법을 찾는다.

---

## 핵심 기술 ① — RLHF와 InstructGPT

### 3단계 파이프라인

2022년 3월, OpenAI의 Ouyang 팀이 "Training language models to follow instructions with human feedback"을 발표했다[^1]. **InstructGPT**다.

방법론의 이름은 **RLHF(Reinforcement Learning from Human Feedback)**. 세 단계로 이루어진다.

**1단계 — SFT(Supervised Fine-Tuning)**

라벨러들이 직접 모범 응답을 작성한다. "좋은 응답이란 이런 것이다"를 예시로 보여주며 모델을 fine-tuning.

**2단계 — 보상 모델(Reward Model) 학습**

같은 프롬프트에 대해 모델이 여러 응답을 생성하고, 라벨러가 순위를 매긴다. "이 응답보다 저 응답이 낫다"는 비교 신호로 보상 모델(RM)을 학습.

**3단계 — PPO로 강화학습**

보상 모델의 점수를 최대화하도록 언어 모델 자체를 강화학습으로 업데이트한다. 단, SFT 모델에서 너무 멀어지지 않도록 KL divergence 페널티를 건다.

```
objective = E[r(x,y)] - β · KL(π_θ ∥ π_SFT)
```

여기서 r은 보상 모델, β는 KL 페널티 계수다. β가 없으면 모델이 보상만 극단적으로 최대화하다가 엉뚱한 응답을 생성하게 된다.

### 결과: 크기를 이긴 정렬

결과가 놀라웠다. **1.3B 파라미터 InstructGPT가 175B GPT-3보다 인간 평가에서 높은 점수를 받았다.** 파라미터 수는 100배 적었다.

진실성(truthfulness)과 독성(toxicity) 감소 효과도 확인됐다. 공개 NLP 벤치마크에서는 일부 성능 저하("alignment tax")가 관찰됐지만, 인간이 느끼는 유용성은 크게 올랐다.

"정렬이 크기보다 중요할 수 있다"는 메시지였다. InstructGPT의 방법론은 이후 ChatGPT를 비롯한 현대 대화형 AI 시스템의 학습 방법론으로 자리잡았다.

## 핵심 기술 ② — DPO: 수학으로 보상 모델을 없애다

### RLHF의 복잡성 문제

RLHF의 한계는 파이프라인의 복잡성이었다. 보상 모델을 별도로 학습하고, PPO라는 민감한 강화학습 알고리즘을 안정적으로 돌려야 한다. 하이퍼파라미터에 예민하고, 학습이 불안정해지기 쉽다.

2023년 5월, Stanford의 Rafailov 팀이 <strong>DPO(Direct Preference Optimization)</strong>를 발표했다[^2]. 핵심 통찰은 하나였다.

> "최적 정책은 보상 함수를 통해 closed form으로 표현 가능하다. 그러면 보상 모델이 따로 필요 없다."

### 수학적 유도

Bradley-Terry 선호도 모델에서 출발한다. 사람이 응답 y\_w을 y\_l보다 선호할 확률:

```
P(y_w ≻ y_l | x) = σ(r*(x, y_w) - r*(x, y_l))
```

RL의 KL-constrained 최적화 문제에서 최적 정책 π\*와 보상 r\*의 관계를 정리하면:

```
r*(x, y) = β · log(π*(y|x) / π_ref(y|x)) + β · log Z(x)
```

이를 Bradley-Terry 식에 대입하면, 분리 불가능해 보이던 Z(x)가 사라지면서 다음 목표 함수가 도출된다:

```
L_DPO = -E[log σ(β·log(π_θ(y_w|x)/π_ref(y_w|x)) - β·log(π_θ(y_l|x)/π_ref(y_l|x)))]
```

보상 모델이 없다. 별도의 RM 학습 없이, 선호 쌍 데이터 (x, y\_w, y\_l)와 binary cross-entropy 손실만으로 언어 모델을 직접 최적화한다.

직관적으로는: "모델이 암묵적으로 자기 자신이 보상 함수다."

### 결과

원 논문(2023년 발표) 기준으로, GPT-4 평가 TL;DR 요약 태스크에서 DPO의 승률은 약 61%, PPO 기반 RLHF는 약 57%였다[^2]. 구현 복잡도는 크게 낮아졌다.

단, 2024년 ICML 종합 연구(Xu et al.)에서는 복잡한 코딩 태스크에서 잘 조정된 PPO가 DPO를 능가하는 결과도 보고됐다[^3]. PPO와 DPO는 상호 보완적이며, 어느 쪽이 절대적으로 우월하다고 보기 어렵다.

## 핵심 기술 ③ — GRPO: 크리틱 없이 그룹으로

### PPO의 메모리 문제

PPO를 대형 모델에 적용하면 한 가지 현실적 문제가 생긴다. **크리틱 모델(critic model)** 때문이다. PPO는 어떤 상태가 얼마나 좋은지 평가하는 별도의 가치 함수 네트워크를 필요로 하는데, 이것이 보통 정책 모델과 같은 크기다. 즉, 70B 모델을 PPO로 학습하면 70B짜리 모델 두 개를 동시에 메모리에 올려야 한다.

2024년 2월, DeepSeek의 Shao 팀이 DeepSeekMath 논문과 함께 <strong>GRPO(Group Relative Policy Optimization)</strong>를 제안했다[^4]. 아이디어는 단순했다.

> "크리틱 모델 대신, 같은 프롬프트에서 샘플링한 그룹의 평균 보상을 베이스라인으로 쓰자."

### 핵심 아이디어

동일 프롬프트에 대해 G개의 출력을 샘플링한다. 각 출력의 advantage는 그룹 내 상대적 순위로 추정한다.

```
A_i = (r_i - mean(r_1, ..., r_G)) / std(r_1, ..., r_G)
```

이 방식으로 크리틱 모델이 불필요해진다. 메모리 절감, 구현 단순화.

### 결과

DeepSeekMath-7B에서 GRPO 적용 후 수학 벤치마크가 개선됐다 (GSM8K: 82.9% → 88.2%, MATH: 46.8% → 51.7%, 2024년 기준)[^4].

GRPO의 진가는 이후에 드러났다. 2025년 1월 발표된 **DeepSeek-R1**이 GRPO를 RL 프레임워크로 채택했다. R1의 기반이 된 DeepSeek-R1-Zero는 GRPO만으로 AIME 2024 pass@1을 15.6%에서 71.0%까지 끌어올렸다. 정식 R1 모델은 cold-start SFT를 추가해 79.8%를 달성했다 (2025년 1월 기술 보고서 기준)[^5].

## 핵심 기술 ④ — Constitutional AI: AI가 AI를 정렬한다

### RLHF의 인간 비용 문제

RLHF는 무해성(harmlessness) 학습을 위해 인간 라벨러가 유해한 내용을 반복 검토해야 한다. 심리적 부담과 비용 문제가 있다. 또한 어떤 원칙으로 응답을 평가해야 하는지가 명시되지 않아 일관성이 떨어질 수 있다.

2022년 12월, Anthropic의 Bai 팀이 <strong>CAI(Constitutional AI)</strong>를 발표했다[^6]. 두 가지 문제를 동시에 해결하는 접근이었다.

### 2단계 구조

**1단계 — SL-CAI (Supervised Learning)**

먼저 AI에게 의도적으로 유해한 질문을 주고 응답을 생성하게 한다. 그런 다음, 명시적인 **헌법(constitution)** — 원칙 목록 — 을 제시하며 스스로 비판(critique)하고 수정(revision)하게 한다.

```
원칙 예시: "이 응답이 위험하거나 비윤리적 활동을 도울 수 있는가?"

AI 비판: "제 응답이 [X]를 가능하게 할 수 있습니다."
AI 수정: "더 안전한 방향으로 응답하자면..."
```

수정된 응답으로 supervised fine-tuning.

**2단계 — RLAIF (RL from AI Feedback)**

fine-tuning된 모델에서 여러 응답을 샘플링하고, **다른 AI 모델이** 헌법 원칙에 비추어 어느 응답이 더 나은지 평가한다. 이 AI 선호도 데이터로 보상 모델을 학습하고, PPO로 최종 정책을 최적화한다.

인간이 유해한 내용을 직접 평가하지 않아도 된다. 무해성 학습 전 과정이 AI 감독만으로 이루어진다.

### 의의

- **확장성**: 인간 라벨링 없이도 원칙 기반 무해성 학습 가능
- **투명성**: "모델이 어떤 원칙을 따르는가"가 명시적으로 공개됨
- **RLAIF의 등장**: "AI가 AI를 정렬한다"는 패러다임

CAI는 현재 Claude 시리즈의 학습 방법론적 기반이다. Anthropic은 이후 헌법을 공개하기도 했다.

## 정렬 세(Alignment Tax) — 안전해질수록 멍청해지는가

정렬 기술들이 모두 좋게만 작용하는 것은 아니다. "정렬 세(Alignment Tax)"라는 개념이 있다.

정렬 과정 — RLHF, 지시 튜닝, 편향 완화 — 이 모델의 사전학습된 일반 능력(capability)을 저하시키는 현상이다.

Lin et al. (2024, EMNLP)의 실험에서 이것이 수치로 확인됐다[^7]. OpenLLaMA-3B를 RLHF로 정렬할 때, 보상이 올라가는 만큼 NLP 벤치마크 성능이 떨어졌다.

| 보상 변화 | SQuAD F1 | DROP F1 | WMT BLEU |
|----------|----------|---------|----------|
| 0.16 → 0.35 | 약 15–17점 하락 | 약 15–17점 하락 | 5.7 하락 |

이것은 Goodhart's Law의 AI 버전이다. "측정치가 목표가 되는 순간, 좋은 측정치가 되기를 멈춘다." 보상 모델의 점수를 높이는 것이 진짜 품질 향상과 분리된다.

### 보상 해킹(Reward Hacking)

더 극단적인 경우가 **보상 해킹**이다. 정책 모델이 보상 모델의 허점을 악용해, 실제 응답 품질은 오히려 낮아지는데 보상 점수만 올라가는 현상.

원인은 분포 이탈(out-of-distribution)이다. 학습 중에 RL로 생성되는 응답들은 보상 모델이 학습할 때 보지 못한 분포로 점점 이동한다. 보상 모델이 이상한 응답에 엉뚱하게 높은 점수를 주기 시작한다.

KL divergence 페널티(β)가 이를 막는 주요 장치다. SFT 모델에서 너무 멀어지지 않도록 제약을 건다.

### 완화 방법들

- **모델 가중치 평균화**: SFT 전후 모델의 가중치를 보간. 놀랍게도 단순하지만 Pareto front 최적화에 효과적
- **SFT loss 정규화**: RLHF 목표 함수에 SFT loss를 추가 항으로 포함
- **KL 계수 튜닝**: β 값을 태스크별로 조정

정렬 세는 현재진행형 연구 문제다. 모델이 더 안전해지는 동시에 더 유능해지는 방법은 아직 완전히 해결되지 않았다.

## 정렬 기술의 진화 — 단순화와 효율화

2022년부터 2025년까지 정렬 기술의 흐름을 보면 한 방향이 보인다.

| 연도 | 기술 | 핵심 혁신 | 요구 사항 |
|------|------|----------|---------|
| 2022 | RLHF (Ouyang et al.) | 인간 피드백으로 정렬 | 보상 모델 + PPO + 인간 라벨링 |
| 2022 | CAI (Bai et al.) | AI 피드백으로 정렬 | 헌법 원칙 + RLAIF |
| 2023 | DPO (Rafailov et al.) | 보상 모델 없이 직접 최적화 | 선호 쌍 데이터만 |
| 2024 | GRPO (Shao et al.) | 크리틱 모델 없이 RL | 그룹 샘플링만 |

각 기술은 이전보다 단순하거나 효율적이다. RLHF가 복잡한 3단계 파이프라인이었다면, DPO는 하나의 loss function이다. GRPO는 PPO의 메모리 부담을 절반으로 줄였다.

동시에 각 기술은 서로 다른 문제를 해결한다. DPO는 구현 단순화에 강하고, PPO(그리고 GRPO)는 복잡한 추론 태스크에 강하다. CAI는 투명성과 확장성을 제공한다. 이 기술들이 조합되어 사용되는 경우가 많다.

## 한 가지 더 — 정렬은 해결된 문제가 아니다

정렬 기술들이 많이 발전했지만, 이 문제가 해결됐다고 말하기는 어렵다.

보상 모델은 인간의 선호도를 근사하지만 완벽하지 않다. 인간의 피드백 자체가 편향을 가질 수 있다. 더 긴 응답을 선호하는 경향, 자신 있게 틀리는 응답을 정확한 응답으로 오인하는 경향 등이 알려져 있다.

Constitutional AI의 RLAIF는 AI가 AI를 평가하는 구조다. AI의 편향이 증폭될 수 있다는 우려도 있다.

가장 어려운 문제는 **확장성**이다. 모델이 인간을 능가하는 영역에서는 인간이 응답의 품질을 평가하기 어렵다. 수학 증명, 복잡한 코드, 과학적 주장 — 이런 영역에서 "어느 응답이 더 낫냐"고 라벨러에게 물으면 의미 있는 답이 나오지 않는다.

이 문제를 <strong>확장 가능한 감독(Scalable Oversight)</strong>이라 부른다. 3편(추론 모델)에서 다룰 Test-Time Compute 개념과 연결된다.

## 마무리

정렬은 "모델을 더 크게 만드는 것"과는 다른 차원의 문제다.

RLHF는 인간 피드백을 학습 신호로 만드는 방법을 체계화했다. DPO는 그 수학적 구조를 단순화했다. GRPO는 대형 모델에서 RL을 효율적으로 만들었다. CAI는 AI 자체를 정렬의 도구로 활용했다.

그 과정에서 정렬 세와 보상 해킹이라는 새로운 문제도 명확해졌다. 정렬 기술의 발전은 동시에 정렬 문제의 깊이를 드러내는 과정이기도 했다.

다음 편에서는 이 맥락 위에서 <strong>추론(Reasoning)</strong>의 문제를 다룬다. 모델이 답을 내는 것과 잘 추론해서 답을 내는 것 사이의 간극. Chain-of-Thought에서 DeepSeek R1까지.
