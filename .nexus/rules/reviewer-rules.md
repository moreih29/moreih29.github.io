# Reviewer Rules

리뷰어가 콘텐츠 리뷰 시 따르는 규칙.

---

## 1. 기술적 정확성

- 수치, 벤치마크, 비교 데이터가 출처와 일치하는지 검증
- 기술 용어 사용이 정확한지 확인
- 출처 태그가 본문과 참고 자료 섹션에서 1:1 대응하는지 확인

### 인용 시스템 검증 (참고자료 컴포넌트)

- [ ] 본문의 모든 `<Ref id={N} />`에 대응하는 `<RefItem id={N} ...>`이 존재하는가
- [ ] `<RefItem>`의 `id` 번호가 본문 참조 순서대로 1~N 연속인가 (중간 빠짐 없음)
- [ ] 각 `<RefItem>`의 `tier` 값이 포닥 브리핑의 tier 분류표와 일치하는가
- [ ] 모든 `<RefItem>`에 `url` 속성이 존재하는가
- [ ] 사용되지 않는 `<RefItem>` (본문에서 참조 안 된 항목)이 없는가
  - 예외: 본문 `<Ref>` 없이 `<RefItem>`만 존재하는 항목은 "추가 참고문헌"으로 허용. 에러로 취급하지 않음

---

## 2. 시각 자산 검증

**조건부 필수 체크** — 포스트가 아래 주제 유형에 해당하면 SVG가 포함되어 있어야 한다:

| 주제 유형 | 필수 이미지 |
|-----------|------------|
| 아키텍처 설명 | 블록 다이어그램 |
| 알고리즘/프로세스 | 플로우차트 |
| 기술 비교 | 비교 도식 |
| 시대별 흐름 | 타임라인 |
| 수식 직관화 | 개념 도식 |

검증 항목:
- [ ] 주제 유형에 해당하는 포스트에 최소 1개 SVG가 포함되어 있는가
- [ ] SVG alt 텍스트가 도식 내용을 서술형으로 설명하는가
- [ ] SVG 저장 경로가 규칙에 맞는가 (시리즈: `public/images/{prefix}/`, 단독: `public/images/{slug}/`)
- [ ] 다크모드 호환 (배경 투명, 텍스트 중간톤)

**SVG 누락 시**: 리뷰 결과에 플래그로 보고. Writer 재작업 요청.

---

## 3. 한국어 문법/문체

- 오탈자, 어색한 표현 확인
- 기존 포스트 톤과의 일관성 검증
- MDX bold 패턴 오류 (`**텍스트)**조사`) 확인

---

## 4. MDX 포맷

- frontmatter 스키마 준수 (title ≤100자, description ≤200자)
- GFM 테이블 문법 정상 여부
- 카테고리가 정의된 값(Web, AI, MLOps, DevOps, General)인지 확인
