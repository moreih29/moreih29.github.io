# Researcher Rules

리서처가 조사 수행 시 따르는 규칙. 객관적 사실 수집과 검증에 집중한다.

**티어 분류는 리서처가 하지 않는다.** 리서처는 출처 유형만 표기하고, 포닥이 티어(T1/T2/T3)를 부여한다.

---

## 1. 출처 수집

### 출처 유형

| 유형 태그 | 출처 유형 |
|----------|----------|
| `paper` | peer-reviewed 논문, 공식 기술 보고서 |
| `blog` | 공식 블로그, 시스템 카드 |
| `conference` | 컨퍼런스 발표, 워크숍 |
| `media` | 기술 미디어, 개인 블로그, 커뮤니티 정리 |

### 수집 원칙

- 핵심 주장에는 가능한 한 peer-reviewed 논문 또는 공식 기술 보고서를 우선 수집
- 비공개 모델 정보: 공식 발표 + 근거 있는 추론 허용. 불확실성은 명시적으로 표기 ("~로 알려져 있다", "~로 추정된다")
- 다수의 독립적 출처가 일관되게 지지하는 추론만 허용
- **각 출처에 URL 필수** — URL 없는 출처는 포닥/라이터가 활용 불가
- **출처별 핵심 내용 1줄 요약** 포함 — 포닥과 라이터가 출처의 역할을 즉시 파악할 수 있도록

### 수치 검증

- **핵심 주장** (성능 비교, 비용, "최초" 클레임): 1차 출처 대조 필수
- **부수적 수치** (파라미터 수, 연도 등): 다수 출처 일치로 충분
- **media 유형의 핵심 수치**: 반드시 교차 검증

---

## 2. 리포트 양식

### 출처 표기 형식

티어 자리를 비워두고 유형을 함께 표기한다. 포닥이 `__`를 `T1`/`T2`/`T3`로 채운다.

```
인라인: ALFWorld에서 34% 향상 [__: arXiv:2210.03629, paper]
하단:   [__: arXiv:2210.03629, paper] Yao et al. (2022). ReAct: ...
```

### 필수 구조

```markdown
# {편} 조사 리포트: {제목}

조사일: {YYYY-MM-DD}

---
## 조사 결과
### {주제별 섹션}
(출처/근거 인라인 포함 — [__: 출처, 유형] 형식)

---
## 출처 목록
- [__: 출처, 유형] 전체 서지 정보
```

### 선택 확장 (해당 시에만 추가)

- **보완 내역**: 보완이 발생하면 반드시 기록 (보완일, 사유, 변경 항목)

### 네이밍

- 시리즈: `{시리즈 prefix}-{편번호}-research.md`
- 단독: `{주제}-research.md`
- 저장 위치: `.nexus/reports/`
